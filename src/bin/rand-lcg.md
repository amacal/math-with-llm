# Linear Congruential Generator

## Overview

The goal of this session is to build a formula that produces a sequence of numbers looking unpredictable, using nothing but arithmetic and a starting value called the seed, with no external source of randomness involved anywhere. The entire sequence is generated by repeatedly applying one fixed rule to the previous value:

$$x_{n+1} = (a \cdot x_n + c) \bmod m$$

Here m is the modulus, which also fixes the size of the state space, since every x_n is confined to the range from 0 up to m minus 1; a is called the multiplier and c the increment, and both are fixed constants chosen once, in advance, alongside m. This construction is called a Linear Congruential Generator, or LCG, and the special case where c is chosen to be zero is traditionally called a Lehmer generator after its originator. Because the formula is completely deterministic, an LCG is a pseudorandom generator rather than a genuine source of randomness: the same seed and constants always reproduce the same sequence, which is exactly what makes it useful for reproducible simulations and unsuitable for cryptographic secrets, a distinction this session returns to below.

## The Hull–Dobell full-period conditions

The most desirable behavior an LCG can have is to visit every single one of its m possible states exactly once before returning to the seed and repeating — this is called achieving full period. Whether a given choice of a and c achieves this was explored by hand for m = 8, trying several pairs from the same starting seed of zero. The pair a = 5, c = 1 produces the sequence 0, 1, 6, 7, 4, 5, 2, 3, and then back to 0, visiting all eight states; but a = 3 with the same c = 1 produces only 0, 1, 4, 5, and then back to 0, a cycle of length four that never touches 2, 3, 6, or 7. Since only the four numbers 1, 3, 5, and 7 are coprime to 8, checking a = 1 and a = 7 as well accounts for every remaining candidate, turning the investigation into an exhaustive check rather than a guess from one example: a = 1 gives full period trivially, while a = 7 collapses immediately into a two-state cycle. Comparing all four outcomes against the quantity a minus 1 showed that full period occurred precisely when 4 divides a minus 1, and separately, changing c from 1 to 2 while keeping the good multiplier a = 5 broke full period as well, collapsing the sequence onto only the even residues, pointing to a second, independent requirement that c and m share no common factor. Both empirical findings are special cases of a classical result, the Hull–Dobell theorem of 1962, which states that an LCG achieves full period from every seed if and only if three conditions hold at once: the greatest common divisor of c and m equals 1, every prime number dividing m also divides a minus 1, and if 4 divides m then 4 must additionally divide a minus 1. For every modulus used here — 8, 256, 65536, and effectively 2^64 — the only prime factor of m is 2, and since each is also divisible by 4, the theorem's second condition is already implied by its third, leaving exactly the two conditions found by hand. That third condition is a fixed requirement on the number 4, not a quantity scaling with the size of m, a distinction that matters once m grows to 2^64 and still only requires checking divisibility by 4.

## Eventual periodicity for any parameters

Even when a and c are chosen badly and full period fails, the sequence never wanders forever without repeating — it always settles into some cycle eventually. The reasoning is the same pigeonhole-and-determinism argument already proved in the Pell's equation session in `gcd-euclidean-pell.md`, where a finite state space combined with a rule that depends only on the current state forces some state to repeat, and a repeated state then forces every subsequent state to repeat identically. Here the state space is simply the m possible values of x_n rather than the (m, d) pairs from that earlier session, but the argument transfers without modification: with only m possible states and infinitely many terms, some state must recur among the first m plus one terms, and the deterministic step formula then reproduces everything that follows it. Hull–Dobell's conditions do not create this periodicity; they only rule out a leftover, non-repeating tail before the cycle begins, guaranteeing the cycle covers all m states from the first seed.

## Entropy and predictability

Since a and c are fixed constants known in advance, and every subsequent x_n is a deterministic function of the seed alone, the entire infinite output stream can carry at most as many bits of genuine unpredictability as the seed itself contains, which is at most log base 2 of m. Worse, whenever the greatest common divisor of a and m equals 1, a modular inverse of a exists modulo m — the exact existence condition already established in `mod-inverse.md` — so the step map itself is invertible: given any x_(n+1), the previous state is recovered as a times that inverse of x_(n+1) minus c, so observing one raw internal state lets an attacker reconstruct the entire sequence in both directions. This is why an LCG, however carefully tuned for full period, suits shuffling a deck or seeding a Monte Carlo simulation, but never cryptographic secrets: concatenating its output bytes cannot produce a number with more genuine entropy than the seed itself carried, however large the concatenated result looks.

## Correctness

The implementation in `rand-lcg.rs` restricts every one of its moduli to a power of two — 2^8, 2^16, and effectively 2^64 — which makes the Hull–Dobell conditions collapse to the two-part check described above. For the modulus 2^64, the constant cannot literally be written as a value of type u64, since u64 only reaches up to 2^64 minus 1, so the implementation uses a sentinel value of zero for its const-generic parameter, meaning "no explicit modulo step is needed at all." This is correct because Rust's wrapping_mul and wrapping_add already discard every bit above position 63 during ordinary u64 arithmetic, and discarding those high bits is mathematically identical to reducing modulo 2^64 exactly: a 64-by-64-bit multiplication produces up to 128 bits of true result, and keeping only the low 64 of them is precisely what taking the remainder after dividing by 2^64 means. The first version of this file instead used the literal 18446744073709551615, which is u64::MAX, or 2^64 minus 1, not 2^64 itself — an off-by-one error that swapped the modulus from an even power of two to an odd number, defeated the free-wraparound optimization by forcing a genuine division through the leftover `% T` step, and made the power-of-two-specific Hull–Dobell reasoning inapplicable to that case. This was caught and fixed during the session by replacing the incorrect literal with the zero-sentinel design above.

## Complexity

Producing one successive value costs O(1) time: a fixed small number of 64-bit word operations — one multiplication, one addition, and for the u8 and u16 cases a modulo by a power of two that compiles to a cheap bitmask — none of which grows as m grows, since m never affects the width of the underlying arithmetic. Space is likewise O(1): the generator's state is exactly three u64 fields, regardless of whether the chosen modulus is 2^8 or 2^64.

## Edge cases

Besides the u64::MAX-versus-2^64 bug described above, the largest modulus in this implementation, 2^64, has no unit test verifying full period by direct enumeration, since iterating through 2^64 states is computationally infeasible; only the 2^8 and 2^16 cases are exhaustively checked in the test suite, leaving the largest case resting on the Hull–Dobell argument alone.

## Worked example

Trace the full-period case by hand: m = 8, a = 5, c = 1, seed x_0 = 0. Each step computes 5 times the current value plus 1, then reduces modulo 8: x_1 = (5*0+1) mod 8 = 1, x_2 = (5*1+1) mod 8 = 6, x_3 = (5*6+1) mod 8 = 31 mod 8 = 7 (since 8*3 = 24 and 31-24 = 7), x_4 = (5*7+1) mod 8 = 36 mod 8 = 4, x_5 = (5*4+1) mod 8 = 21 mod 8 = 5, x_6 = (5*5+1) mod 8 = 26 mod 8 = 2, and x_7 = (5*2+1) mod 8 = 11 mod 8 = 3. One more step gives (5*3+1) mod 8 = 16 mod 8 = 0, landing exactly back on the seed. The full trace, 0, 1, 6, 7, 4, 5, 2, 3, 0, visits every one of the eight possible states exactly once before repeating, matching what the Hull–Dobell theorem predicts: a minus 1 is 4, which is divisible by 4, and c is 1, which shares no common factor with 8.
